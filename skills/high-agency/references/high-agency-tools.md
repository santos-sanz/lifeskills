# High agency daily tools

## Perspective shift: "Who solved this with similar constraints?"
How to apply:
- Find a person, company, or team that solved an analogous problem.
- Extract the decision logic, not surface tactics.
- Adapt to your constraint profile.

Use when:
- You are in novelty panic.
- Team treats the problem as unprecedented.

Failure mode:
- Blind copying without context fit.
Mitigation:
- List 3 differences between your context and the analog before applying.

## Reframe: "How can this be easy?"
How to apply:
- Ask for the lowest-friction path to outcome.
- Remove non-essential quality bars for first iteration.
- Identify one move that creates disproportionate leverage.

Use when:
- Work is overcomplicated or stuck in planning.

Failure mode:
- Oversimplifying critical risk.
Mitigation:
- Keep one explicit risk check before execution.

## Build rapid feedback loops
How to apply:
- Define one leading indicator and one review checkpoint.
- Set a short loop (same day, 24h, or 72h).
- Pre-define pivot conditions.

Use when:
- Uncertainty is high and predictions are weak.

Failure mode:
- Tracking vanity metrics.
Mitigation:
- Choose a metric that changes behavior or allocation decisions.

## Ask for help with high-context requests
How to apply:
- State objective, constraints, and what is already tried.
- Make one clear, bounded ask with deadline.
- Suggest 2-3 response options to reduce cognitive load.

Use when:
- Progress depends on another team, expert, or stakeholder.

Failure mode:
- Vague asks that generate vague answers.
Mitigation:
- Require ask format: context, specific request, due date.

## Increase luck surface area through output frequency
How to apply:
- Increase volume of useful visible output (memos, updates, demos, asks).
- Expand distribution to relevant nodes (partners, customers, operators).
- Keep cadence steady to compound weak ties.

Use when:
- Pipeline, partnerships, hiring, or distribution is thin.

Failure mode:
- Spamming low-quality output.
Mitigation:
- Keep quality floor: each output must contain one concrete value.
