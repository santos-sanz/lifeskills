# Pyramid rules and quality checks

## Core operating rules
- Lead with one answer to one governing question.
- Keep top-level support to 3-5 points.
- Use parallel wording so points are comparable.
- Ensure each level summarizes the level below.

## Logic mode discipline
- Deductive mode: premises lead to conclusion.
- Inductive mode: examples accumulate into conclusion.
- Do not mix deductive and inductive logic at the same level.

## Single governing question test
- Pass if one sentence captures the decision to make.
- Fail if the question contains "and" with multiple decision verbs.
- Fix by splitting into primary and secondary questions; keep only primary.

## Anti-ambiguity checklist
- Replace abstract verbs (optimize, improve) with specific actions.
- Quantify directional claims whenever data exists.
- Separate facts from assumptions explicitly.
- Name owner and date for each next action.

## Evidence discipline
- Mark evidence quality as high, medium, or low confidence.
- Prefer recent internal data for operational decisions.
- If evidence is weak, downgrade certainty in recommendation language.

## Common failure patterns
- Starting with background instead of decision.
- Overlapping support points that restate the same idea.
- Overstuffed sections with no prioritization.
- Recommendations without an accountable owner.
