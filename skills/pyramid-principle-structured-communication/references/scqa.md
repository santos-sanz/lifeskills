# SCQA for executive storyline setup

Use SCQA before drafting the final pyramid when context is noisy.

## Situation
- Capture baseline facts everyone accepts.
- Keep it short; only include context needed for the decision.

## Complication
- Name what changed and why it matters now.
- Focus on one tension that creates decision pressure.

## Question
- Translate complication into a single governing question.
- Phrase as a decision, not a broad exploration.

## Answer
- Provide a direct recommendation with confidence level.
- Add condition if recommendation depends on uncertain inputs.

## Transition to pyramid
- SCQA should generate the BLUF sentence.
- Support points then prove the BLUF using parallel logic.

## Quick quality checks
- If the question is ambiguous, the answer will drift.
- If the complication is weak, urgency will be unclear.
- If situation is too long, the narrative loses executive attention.
